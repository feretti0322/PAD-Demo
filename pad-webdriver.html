🚗 ChromeDriver vs 標準起動 ― どちらを使うべき？

PADにはChromeを起動する方法が2種類あります。本研修では Chrome WebDriver起動 を必修として使います。その理由をここで理解しましょう。

① 2種類の起動方式の比較

項目 標準のChrome起動（CDP方式） Chrome WebDriver起動
PADのアクション名 新しい Chrome を起動する Chrome WebDriverを使用してChromeを起動する
仕組み すでに起動しているChromeに「割り込む」 PADがChromeを子プロセスとして完全制御
セットアップ 不要 ChromeDriverのインストールが必要
業務サイトでの安定性 ⚠️ サイトによって要素が取れないことがある ✅ 業務サイトでも安定して動作しやすい
本研修での推奨 △ 学習初期のみ ◎ 必須（第4回以降）
たとえるなら： 標準起動は「すでに走っている車に後から乗り込む」方式。WebDriver起動は「PADがドライバーとして最初から運転席に座る」方式。後者のほうが、業務サイトという複雑な道では安定して走れます。

② ChromeDriver のインストール手順

  1. Chromeのバージョンを確認する
    アドレスバーに chrome://version を入力 → 「Google Chrome」の行を確認 → メジャーバージョン（例：124）をメモする。
  2. ChromeDriver をダウンロードする
    Chrome 115以降は https://googlechromelabs.github.io/chrome-for-testing/ から、自分のバージョンに対応する chromedriver の win64 をダウンロード。
  3. 展開して所定のフォルダに配置する
    推奨パス：C:\PAD\drivers\chromedriver.exe
  4. PADで設定する
    「Chrome WebDriverを使用してChromeを起動する」アクションを追加し、「ChromeDriverのパス」に C:\PAD\drivers\chromedriver.exe を指定。
起動アクションの設定例： WebDriver: Chrome ChromeDriver パス: C:\PAD\drivers\chromedriver.exe 初期URL: https://example.com/login

③ バージョン不一致に注意

⚠️ 注意： Chromeは自動更新されます。ChromeDriverのバージョンが古くなると、ある日突然フローが動かなくなります。
月に1回、Chromeのバージョンを確認してChromeDriverを更新する運用を推奨します。
典型的な症状：「ChromeDriverとChromeのバージョンが一致しません」というエラーメッセージ。

④ 設定確認チェックリスト

  • C:\PAD\drivers\chromedriver.exe が存在する
  • PADアクションで「ChromeDriverのパス」が正しく設定されている
  • ChromeのバージョンとChromeDriverのメジャーバージョンが一致している
  • Chromeがデフォルトのプロファイルで起動できる

📝 3行まとめ

  1. PADのChrome起動には 標準起動 と WebDriver起動 の2種類があり、業務サイト対応では後者が本命。
  2. ChromeDriver を C:\PAD\drivers\ に配置し、PADアクションでパスを指定すれば準備完了。
  3. Chromeの自動更新でバージョン乖離が起きるため、月次で点検する運用が必要。
