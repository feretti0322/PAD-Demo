🚗 WebDriverと標準起動の違いを学ぼう
👤user

🚗 ChromeDriver vs 標準起動 ― どちらを使うべき？

PADにはChromeを起動する方法が2種類あります。本研修では Chrome WebDriver起動 を必修として使います。その理由をここで理解しましょう。

① 2種類の起動方式の比較

項目 標準のChrome起動（CDP方式） Chrome WebDriver起動
PADのアクション名 新しい Chrome を起動する 新しい Chrome を起動する
仕組み すでに起動しているChromeに「割り込む」 PADがChromeを子プロセスとして完全制御
セットアップ 不要 ChromeDriverのインストールが必要
業務サイトでの安定性 ⚠️ サイトによって要素が取れないことがある ✅ 業務サイトでも安定して動作しやすい
本研修での推奨 △ 学習初期のみ ◎ 必須（第4回以降）
📌 アクション名は同じ、設定で切り替える： PADではどちらも 「新しい Chrome を起動する」 という同じアクションです。アクションの設定画面にある 「操作方法」 プルダウンで 「ブラウザー拡張機能」（標準起動）と 「Webdriver」（WebDriver起動）を切り替えます。
たとえるなら： 標準起動は「すでに走っている車に後から乗り込む」方式。WebDriver起動は「PADがドライバーとして最初から運転席に座る」方式。後者のほうが、業務サイトという複雑な道では安定して走れます。

② ChromeDriver のインストール手順

  1. 手順1：Chromeのバージョンを確認する
    Chrome を開き、アドレスバーに chrome://settings/help と入力する。
    バージョン番号（例: 138.0.7204.169）をメモする。
  2. 手順2：対応する ChromeDriver をダウンロードする
    公式 ChromeDriver ページ（ https://developer.chrome.com/docs/chromedriver ）にアクセスする。
    ダウンロードページで、Chromeブラウザのバージョンと一致するバージョンを選択する（通常、最初の3つの数字が一致していれば問題ない）。
    win-x64 アーキテクチャに適したダウンロードリンクを選択し、ZIP ファイルをダウンロードする。
  3. 手順3：ダウンロードした ZIP ファイルを解凍する
    ダウンロードフォルダに保存された ZIP ファイル（例：chromedriver-win64.zip）を右クリックし、「すべて展開」 を選択する。
    展開先は既定のままで問題ない。展開後、chromedriver-win64 フォルダの中に chromedriver.exe が含まれていることを確認する。
  4. 手順4：Power Automate Desktop 用のフォルダを作成する
    エクスプローラーを開く。
    アドレスバーに以下をコピーして貼り付け、Enter キーを押す：
    %LOCALAPPDATA%\Microsoft\Power Automate Desktop
    フォルダー内で 右クリック → 「新規作成」 → 「フォルダー」 を選択する。
    フォルダー名を WebDrivers にする（大文字・小文字は正確に入力すること）。
  5. 手順5：chromedriver.exe を配置する
    手順3で解凍したフォルダから chromedriver.exe をコピーする。
    手順4で作成した Power Automate Desktop\WebDrivers フォルダーの中に貼り付ける。

    最終的なフォルダー構成は次の通りです：
    AppData └ Local └ Microsoft └ Power Automate Desktop └ WebDrivers └ chromedriver.exe

③ バージョン不一致に注意

⚠️ 注意： Chromeは自動更新されます。ChromeDriverのバージョンが古くなると、ある日突然フローが動かなくなります。
月に1回、Chromeのバージョンを確認してChromeDriverを更新する運用を推奨します。
典型的な症状：「ChromeDriverとChromeのバージョンが一致しません」というエラーメッセージ。

📝 3行まとめ

  1. PADの「新しい Chrome を起動する」アクションは「操作方法」プルダウンで 標準起動と WebDriver起動を切り替える。業務サイト対応では後者が本命。
  2. ChromeDriver は %LOCALAPPDATA%\Microsoft\Power Automate Desktop\WebDrivers\chromedriver.exe に配置すれば PAD が自動で読み込んでくれる。
  3. Chromeの自動更新でバージョン乖離が起きるため、月次で点検する運用が必要。
