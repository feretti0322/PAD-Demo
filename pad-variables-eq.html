🔢 変数（へんすう）って何？

変数とは、データをおきかえるための文字のようなものです。
数学の「x = 5」と同じイメージ！文字に値を代入して使います。

① 変数 ＝ 数学の「文字」と同じ考え方

数学でわからない数字、場合によって変わる数字を「ｘ」にして式を立てたことがありますよね？
プログラミングの変数もまったく同じです。
「文字（変数名）」に「値」を代入して、あとで計算や処理に使います。

担当者名
田中さん
　　
注文数
50
　　
完了フラグ
True
数学では x・y・z のようなシンプルな文字を使いますが、
プログラミングでは 「担当者名」「注文数」 のように意味のある名前をつけます。
PADでは %変数名% と書くことで、代入した値をプログラムの中で使えます。

② なぜ変数を使うの？

数学でも「x に 3 を代入」すれば、x を使った式の答えが全部変わりますよね。
プログラムも同じです。変数の値を1か所変えるだけで、全体に反映されます。

😰 変数を使わないと…

担当者が変わるたびに、プログラムの中の「田中さん」という文字を全部探して書き直す必要があります。書き忘れるとバグの原因に！

メール送信("田中さんへ")
レポート作成("田中さん担当")
通知送信("田中さん確認済")

😊 変数を使うと！

「担当者名 = 田中さん」と代入しておけば、代入値を1か所変えるだけでプログラム全体に反映されます。

担当者名 = "田中さん"
↓
メール送信(%担当者名%へ)
レポート作成(%担当者名%担当)
通知送信(%担当者名%確認済)
数学で「x = 3 → x = 10 に変えると答えも全部変わる」のと同じ考え方です。
変数を使うことで、修正が1か所で済み、ミスも激減します！

③ 変数には「わかりやすい名前」をつけよう

数学では「x」「y」でも通じますが、プログラミングでは中身がわかる名前にするのが鉄則です。
後で見返したときに「この変数、何だっけ？」とならないようにしましょう。

何を代入する変数？ あいまいな名前（NG） わかりやすい名前（OK）
社員の名前 NGa　や　x OK社員名　や　EmployeeName
今月の売上合計 NGn　や　num OK今月売上合計　や　TotalSales
処理が終わったかどうか NGb　や　flag OK処理完了フラグ　や　IsDone
数学の「x」は短くてもOKですが、プログラムは規模が大きくなるほど変数の数も増えます。
「中身が一目でわかる名前」にすることが、ミスを防ぐ一番のコツです！
