🔀 条件分岐とは？ ― 「判断つきの自動化」の入口

「もし入社10年以上のスタッフならベテラン枠でリスト化、それ以外は通常リストに」――こうした判断を組み込めるのが IF / ELSE IF / ELSE アクションです。

① IF / ELSE IF / ELSE の基本構文

IF 勤続年数 >= 10 THEN ベテランリストに追加 ELSE IF 勤続年数 >= 5 THEN 中堅リストに追加 ELSE 新人リストに追加 END
PADでは「If」「Else if」「Else」の3つのアクションを組み合わせて使います。判定対象（変数）と演算子（=, >, <, >=, <=, <>）を選ぶだけのGUI操作です。

② インタラクティブデモ：スタッフ振り分けシミュレーター（IF）

入社日を入力して「判定する」を押すと、勤続年数に応じてベテラン／中堅／新人のいずれかに振り分けられます。

区分条件
ベテラン勤続年数が 10年以上
中堅勤続年数が 5年以上10年未満
新人勤続年数が 5年未満
入社日を選んで「判定する」を押してください

③ Switch の使いどころ

比較する値が1つの変数で、取りうるパターンが複数ある場合は Switch を使うとスッキリ書けます。

SWITCH 部署 CASE "営業部": 営業部用の処理 CASE "総務部": 総務部用の処理 CASE "開発部": 開発部用の処理 DEFAULT: その他の処理 END
IF / ELSE IF を何段も重ねると読みにくくなりがちです。「同じ変数を何パターンかで比較するだけ」という場面では Switch のほうがシンプルに書けます。
ただし、PADの Switch は「完全一致」のみ対応のため、「以上・以下」のような範囲比較はIFを使ってください。

④ インタラクティブデモ：部署振り分けシミュレーター（Switch）

部署を選ぶと、Switch のどの CASE に一致するかがコード上でハイライトされます。

SWITCH 部署 CASE "営業部": 営業部用の処理 CASE "総務部": 総務部用の処理 CASE "開発部": 開発部用の処理 CASE "管理部": 管理部用の処理 DEFAULT: その他の処理 END
部署を選択してください

📝 3行まとめ

  1. IF / ELSE IF / ELSE で「判断つきの自動化」が組める。演算子は = >= <= など。
  2. 同じ変数を複数のパターンと比較する場面では Switch がシンプル。ただし完全一致のみ対応。
  3. 範囲比較（以上・以下）は IF、値のパターン列挙は Switch と使い分けるのがポイント。
