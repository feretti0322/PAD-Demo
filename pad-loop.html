🔁 ループとは？ ― 「全員分やっといて」と一言で済ませる魔法

派遣スタッフ700名分のプロフィールシートを1人ずつ手作業で作るのと、PADに「全員分やっといて」と一言指示して終わらせるのとでは、業務効率が桁違いに変わります。これがループの威力です。

① FOR と FOR EACH の違い

アクション使いどころ例
Loop（FOR） 回数を指定して繰り返す 「メッセージを10回表示する」
For each（FOR EACH） リストやデータテーブルの中身を1件ずつ処理する 「スタッフ一覧の全員に対してプロフィールシートを作る」
覚え方： 数えるのが「回数」なら Loop、数えるのが「件数」なら For each。

② インタラクティブデモ：人間モード vs PADモード

同じ5人分のスタッフ処理を、人間とPADでどれだけ差が出るか体感してみましょう。

👤 人間モード（1人ずつ手作業）
🤖 PADモード（FOR EACH一括）

③ ループとデータテーブルの組み合わせ

スクレイピングで取得したスタッフ一覧（DataTable）を1行ずつ処理する典型パターンです。

FOR EACH CurrentItem IN StaffList 氏名 = CurrentItem['氏名'] 部署 = CurrentItem['部署'] # ↑ 列名はスクレイピング時のヘッダー名と一致させる CALL プロフィールシート出力(氏名, 部署) END

④ ループ内インデックス（LoopIndex）の利用

何件目を処理しているかを LoopIndex で取得できます。進捗ログに便利です。

FOR EACH staff IN StaffList Display message: "%LoopIndex + 1% / %StaffList.RowsCount% 件目: %staff['氏名']%" END

📝 3行まとめ

  1. ループは「同じ処理をたくさん繰り返す」ためのアクション。回数指定なら Loop、件数指定なら For each。
  2. データテーブル × FOR EACH で「スタッフ全員分」のような実務処理が一気に書ける。
  3. 進捗の可視化には LoopIndex を活用すると、運用が安定する。
