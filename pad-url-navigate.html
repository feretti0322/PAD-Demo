🔗 URL直接ナビゲート ― ポップアップを使わずに目的地に直接ワープ

玄関から廊下を通ってリビングに行くのではなく、最初からリビングに直接ワープする方法です。業務サイトでよく遭遇する「PADで要素が取れないUI」を迂回する、実務必須テクニックです。

① 問題提起 ― <object> 埋め込みUIの正体

このデモサイトの「スタッフ検索」ボタン（左サイドバー）を押すと検索ポップアップが表示されます。
しかし、このポップアップは <object> タグで 別ページを埋め込んでいるのです。
PADの「UI要素の追加」でポップアップ内の入力欄を取得しようとしても、取得できません。

実際のコードを見てみましょう。js/common.js の openStaffSearch() 関数の中身：

// js/common.js の openStaffSearch() 抜粋 obj = document.createElement('object'); obj.type = 'text/html'; obj.data = 'staff-search-popup.html'; // ← 別ページを埋め込んでいる document.body.appendChild(obj);

② 観察する ― ポップアップは何をしているか？

諦める前に、ポップアップが最終的に何をしているかを観察しましょう。
staff-search-popup.html の「検索する」ボタンを押すと…
→ staff-list.html?dept=営業部&role=主任 に遷移しています！

staff-search-popup.html の doSearch() 関数を見ると、本当にURLに遷移しているだけだとわかります。

// staff-search-popup.html の doSearch() 抜粋 function doSearch() { const dept = document.getElementById('sDept').value; const role = document.getElementById('sRole').value; const params = new URLSearchParams(); if (dept) params.set('dept', dept); if (role) params.set('role', role); window.top.location.href = 'staff-list.html?' + params.toString(); // ↑ 結局ここに遷移しているだけ！ }

③ 解決策 ― URLパラメータを直接組み立てる

ポップアップを開いて入力する代わりに、PADで URLを直接組み立てて遷移 すれば同じ結果になります。

// ポップアップを操作する代わりに： dept = "営業部" role = "主任" url = "staff-list.html?dept=" + dept + "&role=" + role → Webページに移動する: %url% （= staff-list.html?dept=営業部&role=主任 に直接ナビゲート）
使うアクション：「Webページに移動する」。URLに変数を組み込めば、入力フォームを操作せずに目的の状態にジャンプできます。

④ インタラクティブデモ：URL組み立てシミュレーター

ポップアップと同じ選択肢で部署と役職を選ぶと、生成されるURLがリアルタイムに表示されます。「このURLを開く」を押すと実際に staff-list.html に遷移し、ポップアップを使った場合と同じ結果になることが体感できます。

⑤ URLパラメータを読み解くコツ

業務サイトの検索結果URLには、こんなパターンがよく出てきます。

  • ?page=2 ページ番号の指定
  • ?search=山田 検索キーワードの指定
  • ?dept=営業部&role=主任 複数の絞り込み条件（& で連結）
  • ?id=ST001 特定レコードの直接参照
マッハバイト・バイトル・その他の採用媒体でも、検索結果URLにパラメータが付いていることが多い。知っているかどうかで自動化可能な業務範囲が劇的に変わります。

📝 3行まとめ

  1. <object>/<iframe> の埋め込みUIはPADの要素取得が効かない。
  2. ポップアップが最終的に遷移しているURLを観察し、PADから直接そのURLに行けば同じ結果が得られる。
  3. 業務サイトでも採用媒体でも応用が効く、実務最重要テクニック。
