📦 サブルーティン ― 「1回書いて、何度も使う」

ログイン処理は何度も使う、なら1回書いて使い回せばOK。料理レシピで「だしを取る」が独立しているのと同じ発想で、フローを サブフロー（FUNCTION） として切り出します。

① サブルーティンとは？

PADでは「Sub flow（サブフロー）」がサブルーティンの役割を果たします。Main フローから Run subflow アクションで呼び出せます。

  • 共通処理を1か所にまとめられる（修正も1か所で済む）
  • Main フローが短くなり、全体の流れが読みやすくなる
  • 引数（Input）と戻り値（Output）で値をやり取りできる

② サブルーティンを使わない場合 vs 使う場合

❌ NG例：ログイン処理を3か所にコピペ

[Main] Navigate: index.html Type "user" into ID Type "pass" into PW Click ログイン ... 別の場所 ... Navigate: index.html Type "user" into ID Type "pass" into PW Click ログイン ... さらに別の場所 ... Navigate: index.html Type "user" into ID Type "pass" into PW Click ログイン

パスワード変更時に 3か所すべて 修正が必要。修正漏れの温床。

⭕ OK例：CALL ログイン() で1行に集約

[Main] Run subflow: ログイン ... 別の場所 ... Run subflow: ログイン ... さらに別の場所 ... Run subflow: ログイン [Sub flow: ログイン] Navigate: index.html Type "user" into ID Type "pass" into PW Click ログイン

パスワード変更時は サブフロー1か所 だけ修正すれば全箇所に反映。

③ 引数と戻り値

サブフローは Input variables（引数） と Output variables（戻り値） を定義できます。

[Sub flow: ログイン] Input: userId, password Output: loginSuccess Type %userId% into "ID欄" Type %password% into "PW欄" Click ログインボタン IF エラーメッセージが表示されている THEN loginSuccess = false ELSE loginSuccess = true END [Main] Run subflow: ログイン userId = "user" password = "pass" → 戻り値: loginSuccess IF loginSuccess = false THEN THROW "ログイン失敗"
命名のコツ： サブフロー名は動詞＋目的語（例：「ログインする」「スタッフ情報を取得する」）。何をする処理かが一目でわかります。

④ 最終演習での活用例

第7回の総合演習では、Main フローを以下のようにサブフローに分割します。

[Main フロー] ├─ Run subflow: ログイン処理() ├─ Run subflow: スタッフ一覧取得(部署, 役職) → StaffList └─ FOR EACH staff IN StaffList └─ Run subflow: プロフィールシート出力(staff)
この分割により、「ログインだけ単独テストする」「スタッフ一覧取得だけ単独テストする」といった部品単位の動作確認が可能になります。

📝 3行まとめ

  1. サブフロー（FUNCTION）で「同じ処理を1回書いて何度も呼び出す」が実現できる。
  2. 引数と戻り値で値をやり取り。Main フローは「呼び出しの流れ」だけになり読みやすい。
  3. 実務では「ログイン」「データ取得」「出力」の単位で分割すると、テスト・修正が楽になる。
