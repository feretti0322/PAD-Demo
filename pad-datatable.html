📊 データテーブル ― Excelの表をPADの中で扱える変数

スクレイピング結果やExcelからの読み込み結果は、すべて DataTable 型の変数に格納されます。行と列を持つ表形式のデータを、PADの中で自在に操作できます。

① データテーブルの主な操作

📥

作成・読み込み

スクレイピング結果の格納 / Excelワークシートからの読み込み

➕

行の追加

「データテーブルに行を挿入」アクションで1行ずつ追加

🔍

フィルタ・検索

「データテーブルをフィルター処理」で条件に合う行だけ抽出

📤

書き出し

「Excelワークシートに書き込み」で DataTable をシートに反映

② FOR EACH との組み合わせ

DataTable を1行ずつ取り出す典型パターン。列名は抽出時のヘッダー名と一致させます。

FOR EACH CurrentItem IN StaffList 氏名 = CurrentItem['氏名'] 部署 = CurrentItem['部署'] # ※ 列名はスクレイピング時のヘッダー名と一致させる END
列名 vs インデックス： CurrentItem['氏名'] のような列名指定のほうが、列順が変わってもフローが壊れません。CurrentItem[0] のようなインデックス指定はサイト側の改修に弱い。

📝 3行まとめ

  1. DataTable は「Excelの表」をPADで扱う変数。行と列を持つ。
  2. 主な操作：作成・追加・フィルタ・書き出し。FOR EACH との組み合わせが定番。
  3. セルの取り出しは CurrentItem['列名'] の列名指定が安全。
