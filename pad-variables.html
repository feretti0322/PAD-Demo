🔤 変数を学ぼう
👤user

📦 変数（へんすう）って何？

変数とは、データを入れておくための「箱」のようなものです。
難しそうに聞こえますが、イメージをつかめばすぐわかります！

① 変数 ＝ データを入れておく「箱」

変数は、数字や文字などのデータを一時的に保存しておく場所です。
箱に名前をつけて、中に好きなデータを入れておけます。

箱に名前をつけてデータを入れる
担当者名
田中さん
注文数
50
完了フラグ
True
箱には数字・文字・Yes/No（True/False）など、いろんな種類のデータを入れられます。
PADでは %変数名% と書くことで、箱の中身をプログラムの中で使えます。

② なぜ変数を使うの？

たとえば、毎月の売上レポートを自動で作るプログラムを考えてみましょう。
「今月の担当者名」や「目標金額」が変わるたびにプログラムを書き直すのは大変です。
でも変数を使えば、箱の中身を変えるだけでOKです！

😰 変数を使わないと…

担当者が変わるたびに、プログラムの中の「田中さん」という文字を全部探して書き直す必要があります。書き忘れるとバグの原因に！

メール送信("田中さんへ")
レポート作成("田中さん担当")
通知送信("田中さん確認済")

😊 変数を使うと！

「担当者名」という箱に名前を入れておけば、箱の中身を1か所変えるだけでプログラム全体に反映されます。

担当者名 = "田中さん"
↓
メール送信(%担当者名%へ)
レポート作成(%担当者名%担当)
通知送信(%担当者名%確認済)
変数を使うと、プログラムを1か所変えるだけで全体に反映されます。
書き直しのミスも減り、メンテナンスもかんたんになります！

③ 変数には「わかりやすい名前」をつけよう

箱に名前をつけるとき、「何が入っているかすぐわかる名前」にするのが大切です。
後で見返したときに「この箱、何だっけ？」とならないようにしましょう。

何を入れる箱？ あいまいな名前（NG） わかりやすい名前（OK）
社員の名前 NGa　や　data1 OK社員名　や　EmployeeName
今月の売上合計 NGx　や　num OK今月売上合計　や　TotalSales
処理が終わったかどうか NGflag　や　b OK処理完了フラグ　や　IsDone
名前のルールはプロジェクトやチームによって違いますが、
「中身が想像できる名前」にすることが、ミスを防ぐ一番のコツです！

📝 3行まとめ

  1. 変数はデータを入れておく「箱」のようなもの。PADでは %変数名% と書いて中身を参照する。
  2. 変数を使うと1か所変えるだけでフロー全体に反映でき、書き直しのミスが減る。
  3. 変数名は中身が想像できる名前にすることが、フローを長く使い続けるためのコツ。
