🕷️ スクレイピング ― Webサイトを「読む」自動化

サイトを目で見て手で写す作業の機械化です。PADの 「Webページからのデータ抽出」 アクションで、テーブルや一覧をまるごとデータテーブル変数に取り込めます。

① データ抽出の基本手順

  1. Chrome WebDriver でページを開く
  2. 「Webページからのデータ抽出」アクションを追加
  3. ライブWebヘルパーが起動したら、取得したいテーブル上で右クリック → 「要素全体をHTMLテーブルとして抽出」
  4. 取得結果は DataTable 型変数に格納される（既定名：DataFromWebPage）
  5. FOR EACH でデータテーブルを1行ずつ処理する

② ページネーション（複数ページの走破）

「次へ」ボタンを押しながら全ページを巡回するときは、ライブWebヘルパーで ページャーリンク を指定するか、ループで自前制御します。

LOOP WHILE TRUE Webページからのデータ抽出 → AppendTo: AllData ON BLOCK ERROR (ループを終了) Click: 「次へ」ボタン END END # 「次へ」が無くなった時点でループを抜ける
ライブWebヘルパーの設定で 「ページネーターリンク」 を指定すれば、PADが自動でページ送りをしてくれます。手書きループより楽。

③ <object> / <iframe> 埋め込み時の限界と回避策（必読）

⚠️ 業務サイト最頻出のハマりポイント：
見た目はモーダルなのに、中身が <object> や <iframe> で 別ページを埋め込んでいるUIが実務でよく登場します。このようなUIにPADの「UI要素の追加」をかけると、外側の枠は取れるが 中の入力欄や検索ボタンが取得できません。

本デモサイトの staff-search-popup.html がまさにこのパターンです。サイドバーから「スタッフ検索」を押して開くポップアップは、common.js の openStaffSearch() が <object> 要素として埋め込んでいます。

// common.js の openStaffSearch() 抜粋 obj = document.createElement('object'); obj.type = 'text/html'; obj.data = 'staff-search-popup.html'; // ← 別ページを埋め込み document.body.appendChild(obj);
諦める前に： このような場合の対処法は 「URL直接ナビゲートを学ぼう」 で詳しく学びます。
ポイントは「ポップアップが最終的に何をしているかを観察する」こと。たいていは URLパラメータ付きの画面に遷移しているだけなので、最初からそのURLに直接ジャンプすれば回避できます。

📝 3行まとめ

  1. 「Webページからのデータ抽出」アクションでテーブルを丸ごと DataTable に取り込める。
  2. 複数ページはページネーターリンク指定 or ループで「次へ」をクリックして走破。
  3. <object>/<iframe> の埋め込みUIは要素が取れない → URL直接ナビゲートで回避。
