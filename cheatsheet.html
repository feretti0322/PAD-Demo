⚡ PADチートシート ― すぐ使える定番フロー集

「あのアクション、何だっけ？」というときに開く手元参照用ページ。最終演習・実務で頻出する10パターンを、コピペできる形でまとめています。各パターンの「なぜそうするか」も短く添えました。

📑 目次

🚀 起動・準備 🌐 ブラウザ操作 📊 Excel操作 🔁 ループ・分岐 🛡️ エラー処理 📁 ファイル操作 🎯 ベストプラクティス

🚀 起動・準備

1. Chrome WebDriverでChromeを起動する

Chrome WebDriverを起動 ChromeDriverのパス: C:\PAD\drivers\chromedriver.exe 初期URL: https://◯◯/index.html
業務サイトでは標準起動より安定。第1回で必修。

2. ログイン用のID/PWをグローバル変数に入れておく

%LoginId% = "user" %LoginPw% = "pass"
サブフロー化したログイン処理から参照する。値の変更は1か所で済む。

🌐 ブラウザ操作

3. URLパラメータを組み立てて直接ナビゲートする

%TargetUrl% = "https://◯◯/staff-list.html?dept=" + %Dept% + "&keyword=" + %Keyword% Webページに移動: %TargetUrl%
検索フォームの操作を省略でき、<object> 埋め込みも回避できる。第5回の山場。

4. テーブル全体をスクレイピングする

Webページからデータを抽出 抽出形式: データテーブル 保存先変数: %StaffTable%
ページネーションがあれば「次へ」ボタンと組み合わせて全ページ取得。

📊 Excel操作

5. テンプレートをコピーして別名保存

ファイルのコピー 元: C:\PAD\templates\プロフィールシート_テンプレート.xlsx 先: %OutputFolder%\%StaffId%_%Name%.xlsx
毎回ゼロから作らず、書式が整ったテンプレを使い回す。ファイル名は変数で組み立てる。

6. 特定セルに値を書き込む

Excelワークシートに書き込み セル: B3 値: %StaffName%
セル定義は固定（B2=ID、B3=氏名…）にして、フローを単純化。

🔁 ループ・分岐

7. データテーブルを1行ずつ処理する

For each %CurrentRow% in %StaffTable% %Id% = %CurrentRow['ID']% %Name% = %CurrentRow['氏名']% ... End
%CurrentRow['列名']% で各列の値にアクセスできる。

8. 条件で分岐する

If %YearsOfService% >= 10 then %Tag% = "ベテラン" Else %Tag% = "" End
勤続年数による振り分けなど、判断つきの自動化に必須。

🛡️ エラー処理

9.「全体中断 vs 個別スキップ」の使い分け

ON BLOCK ERROR LogToFile(%error_message%) Continue ← 個別スキップの場合 ※ Throw を使うと全体中断 END
ログイン失敗は全体中断、個別データの欠損は個別スキップ。この使い分けが本研修の山場。
⚠️ 注意：ループ内のエラー処理を「すべてスキップ」にしてしまうと、本来全体中断すべきエラーまで握り潰されます。ログイン失敗系は必ず Throw で再スローしましょう。

📁 ファイル操作

10. 日付フォルダを自動作成して出力先を整理する

%DateStr% = フォーマット(%CurrentDate%, "yyyyMMdd") %OutputFolder% = "C:\Users\%USERNAME%\Desktop\PAD研修_スタッフプロフィール\" + %DateStr% + "\" フォルダの作成: %OutputFolder%
再実行しても上書きされないよう、日付ごとにフォルダを分ける運用。

🎯 ベストプラクティス

変数名はキャメルケース（例：%StaffId%）に統一。複数人で運用するときの可読性が上がる。
ログイン処理・Excel書き込み・エラー記録の3つはサブフロー化してメインフローを短く保つ。
出力先パスはグローバル変数に置く。テスト時に変更しやすい。
WebDriverのセッションはフロー終了時に必ず閉じる。閉じ忘れるとChromeのプロセスが残り続ける。

📝 3行まとめ

  1. 業務サイト自動化はWebDriver起動とURL直接ナビゲートの2点を押さえれば9割解決する。
  2. Excel書き込みは「テンプレコピー → 別名保存 → セル書き込み」の順が定石。
  3. エラーハンドリングは「全体中断か個別スキップか」を最初に決める。これを決めずに書き始めると後で全部書き直しになる。
